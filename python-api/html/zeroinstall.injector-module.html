Package injector

source code

Code relating to interfaces and policies.

To run a program using the injector, the following steps are typical:
  1. Instantiate a policy.Policy (or a sub-class of it), giving it the root interface's URI.
  2. Ask the policy object to choose a set of implementations.
    1. The policy fetches interfaces from the iface_cache, starting a download if needed.
    2. The cached or downloaded XML is parsed into a model using the reader module.
    3. The policy selects a set of implementations suitable for the current arch.
  3. Download the selected implementations and unpack into the zerostore.
  4. Finally, run the program.
For simple command-line use, the autopolicy module provides code to perform the above steps. This is used by the cli module to provide the 0launch interface.
